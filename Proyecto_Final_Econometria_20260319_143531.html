1. Análisis de Regresión Lineal Simple

📋 Modelo: VHT vs Acciones Individuales

Se estimaron cinco modelos de regresión simple para analizar la relación individual entre el ETF VHT y cada acción farmacéutica.

📌 Especificación del Modelo:
VHT_returns = β₀ + β₁ × Stock_returns + ε

Interpretación: El coeficiente β₁ (beta) indica cuánto cambia el retorno de VHT por cada unidad de cambio en el retorno de la acción, asumiendo que los demás factores permanecen constantes.

📊 Diagnósticos de Heterocedasticidad

Se realizaron dos tests para detectar heterocedasticidad:

  • Breusch-Pagan Test: Prueba la hipótesis nula de homocedasticidad
  • Goldfeld-Quandt Test: Complementa el análisis dividiendo la muestra en subgrupos
⚠️ Interpretación: Si el p-valor < 0.05, existe evidencia de heterocedasticidad (varianza no constante de los errores). Esto puede afectar la eficiencia de los estimadores OLS, aunque no los sesgos.

2. Análisis de Regresión Lineal Múltiple

📋 Modelo Múltiple con Todas las Acciones

Se estimó un modelo de regresión múltiple incluyendo los 5 stocks simultáneamente:

📌 Especificación:
VHT = β₀ + β₁×PFE + β₂×ABBV + β₃×MRK + β₄×ABT + β₅×LLY + ε

🔍 Análisis de Multicolinealidad

Se calculó el Variance Inflation Factor (VIF) para cada variable independiente:

  • VIF < 5: Baja multicolinealidad (aceptable)
  • 5 ≤ VIF < 10: Multicolinealidad moderada (puede ser problemática)
  • VIF ≥ 10: Alta multicolinealidad (requiere atención)

⏱️ Análisis de Autocorrelación

Estadístico de Durbin-Watson: Mide correlación serial de residuos

  • DW ≈ 2: Sin autocorrelación
  • DW < 2: Autocorrelación positiva (tendencia)
  • DW > 2: Autocorrelación negativa (inversión)

3. Análisis de Datos de Panel

📊 Estructura de Datos Panel

Características:
  • 5 unidades transversales (stocks)
  • 37 períodos de tiempo (trimestres)
  • Panel balanceado (mismo número de observaciones por stock)
  • Total: 185 observaciones (5 × 37)

🎯 Variable Dependiente

RevenueGrowth: Crecimiento trimestral de ingresos (QoQ %)

🔧 Variables Independientes (Rezagadas 1 Trimestre)

  • GrossMargin_lag1: Margen bruto del trimestre anterior
  • OperatingMargin_lag1: Margen operativo del trimestre anterior
  • ROA_TTM_lag1: Rendimiento sobre activos (TTM) del trimestre anterior
  • AssetTurnover_lag1: Rotación de activos del trimestre anterior

📈 Modelos de Panel Estimados

1️⃣ Pooled OLS

Combina todas las observaciones ignorando la estructura de panel. Asume que todos los stocks tienen el mismo intercepto y coeficientes.

2️⃣ Fixed Effects (Efectos Fijos)

Permite que cada stock tenga su propio intercepto. Controla por características invariantes en el tiempo de cada stock.

Ventajas: Controla heterogeneidad no observada, elimina sesgo de variables omitidas

3️⃣ Random Effects (Efectos Aleatorios)

Asume que los efectos específicos por stock son no correlacionados con los regresores. Más eficiente que FE si el supuesto se cumple.

🔬 Test de Hausman

Compara FE vs RE para determinar cuál modelo es más apropiado:

Hipótesis Nula (H₀): Random Effects es consistente y eficiente
Si p-valor < 0.05: Usar Fixed Effects (RE es inconsistente)
Si p-valor ≥ 0.05: Usar Random Effects (RE es preferible)

📋 Test de Breusch-Pagan LM

Verifica si existe heterogeneidad entre stocks (efectos de panel significativos):

Si p-valor < 0.05: Hay efectos de panel significativos → No usar Pooled OLS
Si p-valor ≥ 0.05: No hay efectos significativos → Pooled OLS puede ser adecuado

4. Análisis de Series de Tiempo y Pronósticos

📈 Variable de Pronóstico: ROA_TTM

Return on Assets (Trailing Twelve Months): Rentabilidad sobre los activos totales en los últimos 12 meses.

🔍 Pruebas de Estacionariedad

Fundamental para modelar series temporales:

  • ADF Test (Augmented Dickey-Fuller): H₀ = Serie tiene raíz unitaria (no estacionaria)
  • KPSS Test: H₀ = Serie es estacionaria
Conclusión: Serie es estacionaria si ADF p-valor < 0.05 Y KPSS p-valor ≥ 0.05

🤖 Modelos de Pronóstico

1. Modelo ARIMA Individual

Se ajusta un modelo ARIMA(1,0,1) para cada stock de forma independiente:

  • AR(1): Un rezago autorregresivo
  • I(0): No diferenciación (serie estacionaria)
  • MA(1): Un término de media móvil

Ventajas: Captura patrones específicos de cada stock. Desventajas: No aprovecha información cruzada.

2. Modelo Panel Autoregresivo (Panel AR)

Regresión con rezagos sobre datos de panel con efectos fijos por ticker:

Especificación:
ROA_TTM = α + β₁×ROA_lag1 + β₂×ROA_lag2 + γ×ROE_lag1 + efectos_fijos + ε

Ventajas: Pooling de información, eficiencia estadística mayor

📊 Métricas de Precisión (In-Sample)

  • MAE: Error Absoluto Medio
  • RMSE: Raíz del Error Cuadrático Medio
  • MAPE: Error Absoluto Porcentual Medio
  • AIC/BIC: Criterios de Información (penalizan complejidad)

🔮 Horizonte de Pronóstico

4 trimestres (1 año) adelante. Apropiado para datos trimestrales con ~37 observaciones por stock.

6. Conclusiones y Recomendaciones

🔑 Hallazgos Principales

  • Relaciones Estadísticas: Las acciones farmacéuticas están altamente correlacionadas con el ETF VHT (como se esperaría dado que el ETF replica el sector healthcare)
  • Heterocedasticidad: Revise los tests de Breusch-Pagan y Goldfeld-Quandt para determinar si hay varianza no constante en los errores
  • Autocorrelación: El estadístico de Durbin-Watson cerca de 2 sugiere baja autocorrelación en series de tiempo financieras
  • Multicolinealidad: Verifique los valores VIF - correlaciones dentro del sector healthcare pueden ser altas
  • Efectos de Panel: El test de Breusch-Pagan determina si usar Pooled OLS o modelos con efectos
  • Hausman Test: Decide entre Fixed Effects y Random Effects basado en la correlación entre efectos y regresores
  • Pronósticos: Compare RMSE y MAPE entre modelos ARIMA y Panel AR para elegir el mejor predictor

✅ Recomendaciones Metodológicas

Para Modelos de Regresión:

  • Si se detecta heterocedasticidad, usar errores estándar robustos (HC1/HC3)
  • Si VIF > 10, considerar eliminar variables o usar regresión ridge
  • Incluir gráficos de residuos para verificar supuestos visualmente

Para Análisis de Panel:

  • Usar Fixed Effects si el test de Hausman es significativo (p < 0.05)
  • Verificar que Random Effects es apropiado si Hausman no es significativo
  • Revisar gráficos de residuos por acción para heterogeneidad

Para Pronósticos:

  • Revisar estacionariedad antes de aplicar ARIMA
  • Usar diferenciación si la serie tiene tendencia
  • Comparar modelos con criterios AIC/BIC y métricas de error
  • Retrain modelos trimestral o mensualmente con datos nuevos

⚠️ Limitaciones y Consideraciones

  • La regresión lineal asume relaciones lineales; los mercados pueden exhibir comportamientos no lineales
  • Los datos históricos no garantizan desempeño futuro
  • Omisión de variables macroeconómicas relevantes (tasas de interés, índices de mercado)
  • Panel data con N=5 (stocks) es relativamente pequeño; más stocks mejoraría la precisión
  • No se incluyen choques estructurales o cambios regulatorios en el sector farmacéutico

🚀 Extensiones Futuras

  • Incluir variables macroeconómicas (tasas, volatilidad, spreads)
  • Modelar volatilidad con GARCH si hay clusters de volatilidad
  • Análisis de cointegración si existe equilibrio de largo plazo
  • Modelos de cambio de régimen (Markov Switching)
  • Machine Learning: Random Forest, XGBoost para pronósticos
